## 1. Overview

R3TRIVE uses a dual-database strategy:

- **SQLite** — default for standalone agent deployments. Zero configuration, single file, embedded in the binary's data directory.
- **PostgreSQL 15+** — for fleet deployments requiring multi-host data, concurrent access, and advanced query capabilities.

The schema is identical between both backends except where noted. An ORM abstraction layer (`internal/storage`) handles dialect differences.

---

## 2. Schema Conventions

### 2.1 ID Format

All primary keys use ULID (Universally Unique Lexicographically Sortable Identifier) format: 26 characters, e.g., `01HN2X4Y5Z6A7B8C9D0E1F2G3H`. ULIDs sort chronologically, making them efficient for time-based queries as primary keys.

### 2.2 Timestamps

All timestamps are stored as `TIMESTAMPTZ` (PostgreSQL) or `TEXT` in ISO 8601 format with nanosecond precision (SQLite). The application layer always works in UTC.

### 2.3 JSON Fields

Complex nested data is stored as JSON. In PostgreSQL, `JSONB` is used for indexable JSON. In SQLite, `TEXT` with JSON format.

### 2.4 Soft Deletes

No hard deletes. All records use `deleted_at` timestamp. Queries include `WHERE deleted_at IS NULL` unless explicitly querying deleted records.

### 2.5 Naming

- Tables: plural, snake_case (`events`, `alert_rules`)
- Columns: snake_case (`created_at`, `host_id`)
- Indexes: `idx_{table}_{columns}` (`idx_events_host_type`)
- Foreign keys: `fk_{table}_{reference}` (`fk_alerts_events`)

## 3. Core Tables

### 3.1 `events`

The central event log. Append-only. Never updated after insert.

```sql
CREATE TABLE events (
    -- Identity
    id              TEXT        NOT NULL,           -- ULID primary key
    
    -- Timing
    timestamp       TIMESTAMPTZ NOT NULL,           -- event occurrence time (nanosecond)
    ingested_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- Classification
    type            TEXT        NOT NULL,           -- e.g., "process.create"
    severity        TEXT        NOT NULL CHECK (severity IN ('low','medium','high','critical')),
    sensor          TEXT        NOT NULL,           -- source sensor name
    
    -- Host reference
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    
    -- Event payload
    data            JSONB       NOT NULL,           -- sensor-specific event data
    enrichments     JSONB       NOT NULL DEFAULT '{}',
    
    -- Integrity
    content_hash    TEXT        NOT NULL,           -- SHA256 of (id + timestamp + type + data)
    chain_hash      TEXT        NOT NULL,           -- SHA256 of (prev_chain_hash + content_hash)
    chain_seq       BIGINT      NOT NULL,           -- monotonic sequence per host
    
    -- Raw capture (optional)
    raw             BYTEA,                          -- base64-encoded raw platform event
    
    PRIMARY KEY (id)
);

-- Partition by timestamp (PostgreSQL only — see §11)
-- SQLite: single table with composite index on (host_id, timestamp)
```

**Key design notes:**
- `chain_hash` and `chain_seq` form the tamper-evident chain (see SYSTEM_ARCHITECTURE.md §4.9.3)
- `data` contains all sensor-specific fields; schema varies by `type`
- Events are never updated after insert (`updated_at` not present)
- Deletions are physical (not soft-delete) only via retention policy enforcement

### 3.2 `alerts`

Alerts generated by the rule engine from events.

```sql
CREATE TABLE alerts (
    -- Identity
    id              TEXT        NOT NULL,           -- ULID
    
    -- Timing
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ,
    
    -- Source
    event_id        TEXT        NOT NULL REFERENCES events(id),
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    rule_id         TEXT        NOT NULL,           -- detection rule ID
    rule_version    TEXT        NOT NULL,           -- rule version at time of alert
    
    -- Classification
    severity        TEXT        NOT NULL CHECK (severity IN ('low','medium','high','critical')),
    confidence      REAL        NOT NULL CHECK (confidence BETWEEN 0 AND 1),
    
    -- ATT&CK mapping
    attack_tactic   TEXT,
    attack_technique TEXT,                          -- e.g., "T1059.001"
    attack_technique_name TEXT,
    
    -- Workflow
    status          TEXT        NOT NULL DEFAULT 'new'
                    CHECK (status IN ('new','acknowledged','investigating','true_positive','false_positive','closed')),
    assignee        TEXT,                           -- user ID or email
    
    -- Context
    context         JSONB       NOT NULL DEFAULT '{}', -- additional alert context
    
    -- Analyst notes
    notes           TEXT,
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_alerts_host_created     ON alerts(host_id, created_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_alerts_status_severity  ON alerts(status, severity) WHERE deleted_at IS NULL;
CREATE INDEX idx_alerts_rule_id          ON alerts(rule_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_alerts_technique        ON alerts(attack_technique) WHERE deleted_at IS NULL;
CREATE INDEX idx_alerts_event_id         ON alerts(event_id);
```

### 3.3 `incidents`

Correlated groups of alerts representing a threat campaign or attack.

```sql
CREATE TABLE incidents (
    -- Identity
    id              TEXT        NOT NULL,           -- INC-YYYYMMDD-NNN format for display
    internal_id     TEXT        NOT NULL,           -- ULID for joins
    
    -- Timing
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    resolved_at     TIMESTAMPTZ,
    deleted_at      TIMESTAMPTZ,
    
    -- Affected scope
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    
    -- Classification
    name            TEXT        NOT NULL,
    severity        TEXT        NOT NULL CHECK (severity IN ('low','medium','high','critical')),
    risk_score      REAL        NOT NULL DEFAULT 0 CHECK (risk_score BETWEEN 0 AND 100),
    
    -- ATT&CK mapping (array of technique IDs)
    attack_techniques TEXT[]    NOT NULL DEFAULT '{}',  -- PostgreSQL; JSON array in SQLite
    
    -- Workflow
    status          TEXT        NOT NULL DEFAULT 'open'
                    CHECK (status IN ('open','investigating','contained','resolved','closed','false_positive')),
    priority        TEXT        CHECK (priority IN ('P1','P2','P3','P4')),
    assignee        TEXT,
    
    -- Content
    summary         TEXT,                           -- Human/AI-generated summary
    ai_analysis     JSONB,                          -- Full AI analysis JSON
    
    -- Response tracking
    containment_at  TIMESTAMPTZ,
    eradication_at  TIMESTAMPTZ,
    
    -- Metadata
    tags            TEXT[]      NOT NULL DEFAULT '{}',
    external_ticket TEXT,                           -- External ticket ID (Jira, SN, etc.)
    
    PRIMARY KEY (internal_id)
);

-- Unique display ID per day (INC-20240315-001, INC-20240315-002, ...)
CREATE UNIQUE INDEX idx_incidents_display_id ON incidents(id) WHERE deleted_at IS NULL;
CREATE INDEX idx_incidents_host_status       ON incidents(host_id, status) WHERE deleted_at IS NULL;
CREATE INDEX idx_incidents_risk_score        ON incidents(risk_score DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_incidents_created_at        ON incidents(created_at DESC);
```

### 3.4 `incident_alerts`

Many-to-many junction between incidents and alerts.

```sql
CREATE TABLE incident_alerts (
    incident_id     TEXT        NOT NULL REFERENCES incidents(internal_id),
    alert_id        TEXT        NOT NULL REFERENCES alerts(id),
    added_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    is_primary      BOOLEAN     NOT NULL DEFAULT FALSE,  -- the triggering alert

    PRIMARY KEY (incident_id, alert_id)
);

CREATE INDEX idx_incident_alerts_alert ON incident_alerts(alert_id);
```

### 3.5 `incident_timeline`

Ordered record of all events in an incident's lifecycle.

```sql
CREATE TABLE incident_timeline (
    id              TEXT        NOT NULL,           -- ULID
    incident_id     TEXT        NOT NULL REFERENCES incidents(internal_id),
    timestamp       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    entry_type      TEXT        NOT NULL
                    CHECK (entry_type IN (
                        'alert_added', 'status_change', 'analyst_note',
                        'response_action', 'ai_analysis', 'evidence_added',
                        'assignee_change', 'priority_change'
                    )),
    
    actor           TEXT,                           -- user ID or "system"
    summary         TEXT        NOT NULL,
    details         JSONB       NOT NULL DEFAULT '{}',
    
    -- References to related objects
    alert_id        TEXT        REFERENCES alerts(id),
    action_id       TEXT        REFERENCES response_actions(id),
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_timeline_incident ON incident_timeline(incident_id, timestamp ASC);
```

## 4. Host Tables

### 4.1 `hosts`

Registry of all monitored hosts.

```sql
CREATE TABLE hosts (
    id              TEXT        NOT NULL,           -- ULID, generated on first contact
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_seen_at    TIMESTAMPTZ,
    deleted_at      TIMESTAMPTZ,
    
    -- Identity
    hostname        TEXT        NOT NULL,
    fqdn            TEXT,
    
    -- OS information
    os              TEXT        NOT NULL CHECK (os IN ('linux','windows','macos','freebsd','other')),
    os_version      TEXT,
    os_build        TEXT,
    arch            TEXT        NOT NULL,           -- amd64, arm64, 386
    
    -- Agent info
    agent_version   TEXT,
    agent_status    TEXT        NOT NULL DEFAULT 'unknown'
                    CHECK (agent_status IN ('online','offline','degraded','unknown')),
    
    -- Network
    primary_ip      TEXT,
    mac_address     TEXT,
    
    -- Classification
    tags            TEXT[]      NOT NULL DEFAULT '{}',
    asset_criticality TEXT      DEFAULT 'medium'
                    CHECK (asset_criticality IN ('low','medium','high','critical')),
    environment     TEXT,                           -- production, staging, dev
    owner           TEXT,                           -- team or user responsible
    
    -- Sensor status
    sensor_status   JSONB       NOT NULL DEFAULT '{}',  -- per-sensor health
    
    -- Risk
    current_risk_score REAL     NOT NULL DEFAULT 0,
    
    PRIMARY KEY (id)
);

CREATE UNIQUE INDEX idx_hosts_hostname ON hosts(hostname) WHERE deleted_at IS NULL;
CREATE INDEX idx_hosts_status         ON hosts(agent_status) WHERE deleted_at IS NULL;
CREATE INDEX idx_hosts_tags           ON hosts USING GIN(tags);  -- PostgreSQL only
```

### 4.2 `host_metrics`

Time-series resource utilization from agents.

```sql
CREATE TABLE host_metrics (
    id              TEXT        NOT NULL,
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    timestamp       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- Agent metrics
    cpu_percent     REAL,
    memory_mb       REAL,
    events_per_sec  REAL,
    alerts_per_min  REAL,
    queue_depth     INTEGER,
    
    -- Storage metrics
    db_size_mb      REAL,
    event_count     BIGINT,
    
    PRIMARY KEY (id)
);

-- Retained for 30 days; older data aggregated to hourly buckets
CREATE INDEX idx_host_metrics_host_time ON host_metrics(host_id, timestamp DESC);
```

## 5. Intelligence Tables

### 5.1 `iocs`

Indicators of Compromise from all sources.

```sql
CREATE TABLE iocs (
    id              TEXT        NOT NULL,           -- ULID
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ,
    deleted_at      TIMESTAMPTZ,
    
    -- IOC data
    type            TEXT        NOT NULL
                    CHECK (type IN ('ip','domain','url','md5','sha1','sha256','sha512',
                                    'cert_thumbprint','email','user_agent','registry_key')),
    value           TEXT        NOT NULL,
    value_normalized TEXT       NOT NULL,           -- lowercase, stripped, canonicalized
    
    -- Context
    confidence      INTEGER     NOT NULL DEFAULT 60 CHECK (confidence BETWEEN 0 AND 100),
    severity        TEXT        NOT NULL DEFAULT 'medium'
                    CHECK (severity IN ('low','medium','high','critical')),
    
    -- Provenance
    source          TEXT        NOT NULL,           -- manual, misp, taxii, ai-extracted, etc.
    feed_id         TEXT        REFERENCES threat_feeds(id),
    external_id     TEXT,                           -- IOC ID in originating system
    
    -- Labeling
    tags            TEXT[]      NOT NULL DEFAULT '{}',
    malware_family  TEXT,
    threat_actor    TEXT,
    campaign        TEXT,
    
    -- Usage
    match_count     INTEGER     NOT NULL DEFAULT 0,
    last_matched_at TIMESTAMPTZ,
    
    -- Analyst
    notes           TEXT,
    added_by        TEXT,
    
    PRIMARY KEY (id)
);

-- Fast lookups by normalized value
CREATE UNIQUE INDEX idx_iocs_type_value  ON iocs(type, value_normalized) WHERE deleted_at IS NULL AND expires_at > NOW();
CREATE INDEX idx_iocs_type               ON iocs(type) WHERE deleted_at IS NULL;
CREATE INDEX idx_iocs_expires_at         ON iocs(expires_at);
CREATE INDEX idx_iocs_tags               ON iocs USING GIN(tags);  -- PostgreSQL only
```

### 5.2 `ioc_matches`

Record of every IOC match event.

```sql
CREATE TABLE ioc_matches (
    id              TEXT        NOT NULL,
    ioc_id          TEXT        NOT NULL REFERENCES iocs(id),
    event_id        TEXT        NOT NULL REFERENCES events(id),
    alert_id        TEXT        REFERENCES alerts(id),
    matched_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    matched_field   TEXT        NOT NULL,           -- which event field matched
    matched_value   TEXT        NOT NULL,           -- the actual value that matched
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_ioc_matches_ioc    ON ioc_matches(ioc_id, matched_at DESC);
CREATE INDEX idx_ioc_matches_event  ON ioc_matches(event_id);
CREATE INDEX idx_ioc_matches_host   ON ioc_matches(host_id, matched_at DESC);
```

### 5.3 `threat_feeds`

Configuration and status of external threat intelligence feeds.

```sql
CREATE TABLE threat_feeds (
    id              TEXT        NOT NULL,
    name            TEXT        NOT NULL,
    type            TEXT        NOT NULL CHECK (type IN ('taxii','misp','stix','csv','json','custom')),
    url             TEXT        NOT NULL,
    enabled         BOOLEAN     NOT NULL DEFAULT TRUE,
    
    -- Auth (encrypted at rest)
    auth_type       TEXT        CHECK (auth_type IN ('none','api_key','basic','oauth2','mtls')),
    auth_config     BYTEA,                          -- AES-GCM encrypted JSON
    
    -- Sync
    sync_interval   INTERVAL    NOT NULL DEFAULT '1 day',
    last_sync_at    TIMESTAMPTZ,
    last_sync_status TEXT,
    ioc_count       INTEGER     NOT NULL DEFAULT 0,
    
    -- Config
    config          JSONB       NOT NULL DEFAULT '{}',
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    PRIMARY KEY (id)
);
```

## 6. Rule Tables

### 6.1 `rules`

All detection rules (atomic, correlation, aggregate, suppression).

```sql
CREATE TABLE rules (
    id              TEXT        NOT NULL,           -- e.g., "R3T-EXEC-001"
    version         TEXT        NOT NULL,
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ,
    
    -- Content
    name            TEXT        NOT NULL,
    description     TEXT,
    type            TEXT        NOT NULL CHECK (type IN ('atomic','sequence','aggregate','suppression')),
    status          TEXT        NOT NULL DEFAULT 'stable'
                    CHECK (status IN ('draft','experimental','stable','deprecated')),
    enabled         BOOLEAN     NOT NULL DEFAULT TRUE,
    
    -- Rule definition (YAML stored as text)
    content         TEXT        NOT NULL,
    content_hash    TEXT        NOT NULL,
    
    -- Metadata
    author          TEXT,
    tags            TEXT[]      NOT NULL DEFAULT '{}',
    references      TEXT[]      NOT NULL DEFAULT '{}',
    
    -- ATT&CK
    attack_tactic   TEXT,
    attack_technique TEXT,
    
    -- Settings
    severity        TEXT        NOT NULL CHECK (severity IN ('low','medium','high','critical')),
    confidence      REAL        NOT NULL CHECK (confidence BETWEEN 0 AND 1),
    
    -- Source
    source          TEXT        NOT NULL DEFAULT 'builtin'
                    CHECK (source IN ('builtin','custom','community','ai-generated')),
    
    PRIMARY KEY (id, version)
);

-- Current active version of each rule
CREATE UNIQUE INDEX idx_rules_active ON rules(id) 
    WHERE deleted_at IS NULL AND enabled = TRUE AND status IN ('experimental','stable');
```

### 6.2 `rule_stats`

Performance and accuracy statistics per rule.

```sql
CREATE TABLE rule_stats (
    rule_id         TEXT        NOT NULL,
    date            DATE        NOT NULL DEFAULT CURRENT_DATE,
    
    -- Volume
    alerts_generated INTEGER    NOT NULL DEFAULT 0,
    
    -- Accuracy (requires analyst feedback)
    true_positives   INTEGER    NOT NULL DEFAULT 0,
    false_positives  INTEGER    NOT NULL DEFAULT 0,
    
    -- Performance
    avg_eval_us      REAL,                          -- average evaluation time (microseconds)
    
    PRIMARY KEY (rule_id, date)
);
```

### 6.3 `suppressions`

False positive suppression rules. Auto-generated or analyst-defined.

```sql
CREATE TABLE suppressions (
    id              TEXT        NOT NULL,
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ NOT NULL,
    deleted_at      TIMESTAMPTZ,
    
    -- Scope
    rule_id         TEXT        NOT NULL,           -- rule to suppress
    
    -- Match conditions (must all match for suppression to apply)
    conditions      JSONB       NOT NULL,
    
    -- Context
    created_by      TEXT        NOT NULL,
    original_alert_id TEXT      REFERENCES alerts(id),
    note            TEXT,
    
    -- Stats
    suppressed_count INTEGER    NOT NULL DEFAULT 0,
    last_triggered_at TIMESTAMPTZ,
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_suppressions_rule ON suppressions(rule_id) 
    WHERE deleted_at IS NULL AND expires_at > NOW();
```

## 7. Response Tables

### 7.1 `response_actions`

Record of all response actions taken (automated or manual).

```sql
CREATE TABLE response_actions (
    id              TEXT        NOT NULL,
    
    executed_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    completed_at    TIMESTAMPTZ,
    
    -- Context
    incident_id     TEXT        REFERENCES incidents(internal_id),
    alert_id        TEXT        REFERENCES alerts(id),
    host_id         TEXT        NOT NULL REFERENCES hosts(id),
    
    -- Action definition
    action_type     TEXT        NOT NULL
                    CHECK (action_type IN (
                        'kill_process','block_ip','quarantine_file','isolate_host',
                        'disable_account','kill_connection','disable_service',
                        'revoke_token','plugin_action','playbook_step'
                    )),
    params          JSONB       NOT NULL,
    
    -- Execution
    status          TEXT        NOT NULL DEFAULT 'pending'
                    CHECK (status IN ('pending','running','success','failed','rolled_back')),
    error_message   TEXT,
    result          JSONB,
    
    -- Attribution
    executed_by     TEXT        NOT NULL,           -- user ID or "system"
    dry_run         BOOLEAN     NOT NULL DEFAULT FALSE,
    
    -- Rollback
    rollback_id     TEXT,                           -- self-reference for rollback chain
    is_rollback     BOOLEAN     NOT NULL DEFAULT FALSE,
    rollback_of     TEXT        REFERENCES response_actions(id),
    
    -- Audit
    note            TEXT,
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_actions_incident  ON response_actions(incident_id, executed_at DESC);
CREATE INDEX idx_actions_host      ON response_actions(host_id, executed_at DESC);
CREATE INDEX idx_actions_status    ON response_actions(status) WHERE status IN ('pending','running');
```

### 7.2 `playbooks`

Response playbook definitions.

```sql
CREATE TABLE playbooks (
    id              TEXT        NOT NULL,
    version         TEXT        NOT NULL,
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ,
    
    name            TEXT        NOT NULL,
    description     TEXT,
    enabled         BOOLEAN     NOT NULL DEFAULT TRUE,
    
    -- Trigger conditions
    trigger_conditions JSONB    NOT NULL,
    
    -- Playbook steps (ordered)
    steps           JSONB       NOT NULL,
    
    -- Metadata
    author          TEXT,
    tags            TEXT[]      NOT NULL DEFAULT '{}',
    
    PRIMARY KEY (id, version)
);
```

### 7.3 `playbook_executions`

History of playbook executions.

```sql
CREATE TABLE playbook_executions (
    id              TEXT        NOT NULL,
    playbook_id     TEXT        NOT NULL,
    playbook_version TEXT       NOT NULL,
    
    started_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    completed_at    TIMESTAMPTZ,
    
    incident_id     TEXT        REFERENCES incidents(internal_id),
    triggered_by    TEXT        NOT NULL,           -- "auto" or user ID
    
    status          TEXT        NOT NULL DEFAULT 'running'
                    CHECK (status IN ('running','success','partial','failed','rolled_back')),
    
    steps_executed  JSONB       NOT NULL DEFAULT '[]',
    dry_run         BOOLEAN     NOT NULL DEFAULT FALSE,
    
    PRIMARY KEY (id)
);
```

## 8. AI Tables

```sql
CREATE TABLE ai_requests (

                    )),

    requested_by    TEXT,
    
    -- Backend
    backend_used    TEXT        NOT NULL,
    model_used      TEXT        NOT NULL,

    response_summary TEXT,

CREATE INDEX idx_ai_requests_incident ON ai_requests(incident_id);

```

### 8.2 `ai_feedback`

Analyst ratings on AI responses.

```sql
CREATE TABLE ai_feedback (

    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    submitted_by    TEXT        NOT NULL,
    
    rating          INTEGER     NOT NULL CHECK (rating BETWEEN 1 AND 5),
    note            TEXT,
    
    -- Structured feedback
    accurate        BOOLEAN,
    actionable      BOOLEAN,
    hallucinated    BOOLEAN,
    
    PRIMARY KEY (id)
);
```

## 9. Auth and Audit Tables

### 9.1 `users`

User accounts (for API server / fleet controller mode).

```sql
CREATE TABLE users (
    id              TEXT        NOT NULL,
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ,
    
    username        TEXT        NOT NULL,
    email           TEXT        NOT NULL,
    
    -- Auth
    password_hash   TEXT,                           -- bcrypt
    mfa_secret      BYTEA,                          -- encrypted TOTP secret
    mfa_enabled     BOOLEAN     NOT NULL DEFAULT FALSE,
    
    -- Authorization
    role            TEXT        NOT NULL DEFAULT 'analyst'
                    CHECK (role IN ('viewer','analyst','responder','admin')),
    
    -- Status
    active          BOOLEAN     NOT NULL DEFAULT TRUE,
    last_login_at   TIMESTAMPTZ,
    failed_login_count INTEGER  NOT NULL DEFAULT 0,
    locked_until    TIMESTAMPTZ,
    
    PRIMARY KEY (id)
);

CREATE UNIQUE INDEX idx_users_username ON users(username) WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX idx_users_email    ON users(email) WHERE deleted_at IS NULL;
```

### 9.2 `api_keys`

API keys for programmatic access.

```sql
CREATE TABLE api_keys (
    id              TEXT        NOT NULL,
    
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ,
    deleted_at      TIMESTAMPTZ,
    last_used_at    TIMESTAMPTZ,
    
    user_id         TEXT        NOT NULL REFERENCES users(id),
    name            TEXT        NOT NULL,
    
    -- Key (stored as hash; full key only shown on creation)
    key_prefix      TEXT        NOT NULL,           -- first 8 chars (for display)
    key_hash        TEXT        NOT NULL,           -- SHA256 of full key
    
    role            TEXT        NOT NULL,           -- inherits max role from user
    scopes          TEXT[]      NOT NULL DEFAULT '{}',  -- optional scope restrictions
    
    PRIMARY KEY (id)
);

CREATE INDEX idx_api_keys_hash   ON api_keys(key_hash) WHERE deleted_at IS NULL;
CREATE INDEX idx_api_keys_user   ON api_keys(user_id);
```

### 9.3 `audit_log`

Immutable audit log of all user and system actions.

```sql
CREATE TABLE audit_log (
    id              TEXT        NOT NULL,
    timestamp       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- Actor
    actor_type      TEXT        NOT NULL CHECK (actor_type IN ('user','system','api_key','plugin')),
    actor_id        TEXT        NOT NULL,
    actor_ip        TEXT,
    
    -- Action
    action          TEXT        NOT NULL,           -- e.g., "alert.status_update"
    resource_type   TEXT        NOT NULL,           -- e.g., "alert"
    resource_id     TEXT,
    
    -- Details
    changes         JSONB,                          -- before/after for updates
    metadata        JSONB       NOT NULL DEFAULT '{}',
    
    -- Result
    success         BOOLEAN     NOT NULL,
    error_message   TEXT,
    
    PRIMARY KEY (id)
);

-- Audit log is append-only; no updates or deletes permitted
-- Enforced via PostgreSQL row-level security or SQLite trigger

CREATE INDEX idx_audit_log_actor     ON audit_log(actor_id, timestamp DESC);
CREATE INDEX idx_audit_log_resource  ON audit_log(resource_type, resource_id, timestamp DESC);
CREATE INDEX idx_audit_log_timestamp ON audit_log(timestamp DESC);
```

## 11. Partitioning Strategy

### 11.1 PostgreSQL Table Partitioning

The `events` table is range-partitioned by day and hash-partitioned by `host_id`:

```sql
-- Parent table (no data stored here)
CREATE TABLE events (
    ...
) PARTITION BY RANGE (timestamp);

-- Daily partitions created automatically by maintenance job
CREATE TABLE events_2024_03_15 
    PARTITION OF events
    FOR VALUES FROM ('2024-03-15') TO ('2024-03-16');
```

Partition management:
- New partitions created 7 days in advance
- Old partitions detached (not deleted) after retention period
- Detached partitions archived to cold storage (S3/GCS) via `pg_dump` → compressed file
- Partition table statistics updated nightly

### 11.2 Retention Periods

| Table | Default Retention | Configurable |
|---|---|---|
| `events` | 30 days (SQLite), 365 days (PostgreSQL) | Yes |
| `alerts` | 365 days | Yes |
| `incidents` | 5 years | Yes |
| `audit_log` | 7 years | Yes (compliance) |
| `host_metrics` | 30 days (raw), 1 year (hourly aggregate) | Yes |
| `ai_requests` | 90 days | Yes |
| `plugin_events` | 7 days | Yes |
| `ioc_matches` | 365 days | Yes |

## 13. Migration Strategy

R3TRIVE uses sequential numbered migrations managed by a built-in migration runner.

```
/internal/storage/migrations/
├── 00001_initial_schema.up.sql
├── 00001_initial_schema.down.sql
├── 00002_add_ai_tables.up.sql
├── 00002_add_ai_tables.down.sql
└── ...
```

Migration conventions:
- Every migration has an `up` and `down` file
- Migrations are transactional (rollback on failure)
- Schema version tracked in `schema_migrations` table
- Migrations run automatically on startup (flag: `--migrate=auto|manual|off`)
- Never modify existing migrations; always add new ones

## 14. SQLite vs PostgreSQL Differences

| Feature | SQLite | PostgreSQL |
|---|---|---|
| JSON type | `TEXT` (JSON string) | `JSONB` (binary, indexable) |
| Arrays | `TEXT` (JSON array string) | Native `TEXT[]` with GIN index |
| Timestamps | `TEXT` ISO 8601 | `TIMESTAMPTZ` |
| Partitioning | Single table + composite index | Native table partitioning |
| Concurrent writes | Single writer | Full concurrent writes |
| Full-text search | FTS5 virtual table | `tsvector` + GIN index |
| CIDR type | Text + application logic | Native `CIDR`/`INET` type |
| Encryption at rest | SQLCipher (optional) | pgcrypto / TDE (OS-level) |
| Max recommended size | 50GB | Unlimited |
| Replication | None (file copy only) | Streaming replication |
